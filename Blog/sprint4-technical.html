The Dreaded Error Message

In one of my tasks I needed do some work with objects. Objects in code are similar to objects in real life in the sense that you refer to them by a name and associate certain preperties with them. If you made an object to represent a ball, you might call it - 'ballObj' and give it a property - 'shape' with the value - 'round'.

Now in most cases in code you use = to signify you want to tie a value to what is called a variable (eg. round), a container for that value (eg. shape). But in the case of objects you use something different called key-value pairs. These make it easier to do certain things with objects that I wont get into but just know that with objects you dont use = to link the two together you use :.

This slipped my mind while working away and when I came to test my code... Errors ensued. On first glance it seemed my code was all written correctly. I wasn't missing any brackets - ( ) or semi-colons - ;, which are the usual culprates.

So I checked out what the error messages had to say "Uncaught SyntaxError: Invalid shorthand property initializer". It was telling me I'd used the wrong syntax = instead of : when writing my code. Oops. Quick fix, just had to change the one character and my code worked as intended.

I let the differences in object syntax slip my mind while coding and was genuinly stumped as to why my code wouldn't run. But by checking out the error message it became clear where I'd gone wrong. This was a good reminder to myself and is to others, that even if you've learnt something in the past, you can't gaurentee you'll get it right everytime from then on out, we all make mistakes. But taking the time to figure out where you went wrong can help lessen the chances you'll make it again in the future. And remember - The Error Message might seem daunting but it often has some great insight into what's gone wrong.

Elegant Solutions

There was a function I needed to make that could take a string and capitalize the first letter of every word. To the best of my knowledge JavaScript doesnt have a built in way to do this, however there is a function that will capitalize everything you give it. So I figured I needed a way to pass that function .toUpperCase() only the first letter of all the words in my sentence.

I managed to do this by taking the sentence (stored as a string) and using .split(" ") ,creating an array from the string with each word (detected by the presence of a blank space) in its own cell. From here I ran a function taking the first character in each cell of the array cell[0], capitalizing it using the .toUpperCase() function and then pairing it back together using string concatenation "H" + "ello" -> "Hello" with the respective word minus that first character using cell.splice(1). All of that was controlled by a map which allows you to call functions on each item in a collection, which in this case was the array made from the original string. After all the words (cells) had been run through they were joined back into a string again using the .join(" ") function, connecting the arrays cells with a sapace be each.

The process ran like this:
"hello world" -> ["hello", "world"] -> H + ello & W + orld -> ["Hello", "World"] -> "Hello World"

This is the actual code I wrote: 

          function capitalize (str){ 
            return str.split(" ").map(x => x[0].toUpperCase() + x.slice(1)).join(" ");
          }
